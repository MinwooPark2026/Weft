# conti-qa — 콘티 품질 검토 (기계 검증 다음 단계)

`weft conti`를 **위반 0건으로 통과한 CONTI.md**가 입력이다. 통과 전이면 먼저
`cd <project> && weft conti`로 0건을 만들고 온다. 이 스킬은 그 위에서
**취향·연출·안전성 lint**를 한다 — 콘티가 "유효"한지가 아니라 "좋은지"를 본다.

> ⚠️ **기계 검증과 중복 금지.** `weft/validate.py`가 이미 잡는 것은 보고하지 않는다:
> 비트 커버리지 빈틈·중복(I1·I3), cover 범위·pause 끝점(I2·I9), 몽타주 인덱스·weight(I4·I5),
> reuse 대상·체인(I6), PICKS 정합(I7·I8·I10), 시간 열 연속성(I11), 15초 초과 단일 이벤트
> 페이스 경고(I12). 이걸 다시 검사하면 소음이다.

검토 기준은 [`script-to-conti/SKILL.md`](../script-to-conti/SKILL.md)의 규칙이다 — 이 스킬이
만든 콘티를 그 스킬의 약속대로 만들어졌는지 확인하는 것. 대본 주제는 불문(분석·사고실험·
비교·음악사 등). **대본의 문장·주장은 고치지 않는다** — 의심 항목은 사용자에게 표시만 한다.

## 입력

- `<project>/CONTI.md` (콘티 표 + shot 표)
- `<project>/CARDS.json` (❝ shot이 있을 때 — CONTI.md 옆이 정본, `weft conti`가 generated_project로 복사)
- 스타일 바이블: CONTI.md 머리말의 `스타일 바이블:` 줄 (없으면 STYLE.txt / DEFAULT_STYLE)

## 체크리스트

### 1. 리듬 (기준 페이스: 장면 5~6초)

- **평균 장면 길이**: `total_seconds ÷ 비디오 이벤트 수`가 **7초를 넘으면** 경고 — 전문가
  기준으로 10초/장면은 속도감이 없다. 기본 목표는 **5~6초/장면**(빠른 페이스는 4초).
- **한 shot 홀드 과다**: 한 shot의 cover 구간이 **약 15초 또는 3비트 이상**이면 시각 정체
  의심(⤴ 단계가 중간에 있으면 단계 사이 구간 기준). 중간에 `▶` 분할·⤴ 빌드업·`▦`를 제안.
  단 15초 초과는 기계 검증 I12가 이미 warning으로 잡는다 — 여기서는 **12~15초의 경계 구간**과
  "I12에 안 걸리지만 연속으로 정체된 패턴"을 본다(중복 보고 금지).
- **단조 패턴**: 빠른 페이스에서 연속 `▶` 자체는 정상이다. 대신 **10비트 이상 같은 류의
  전환만 반복**(전부 ▶ 단신, 홀드·몽타주·⏸·↺ 전무)이면 리듬 단조 경고 — 편집의 차원이 없다.
- **몽타주 과밀**: `▦` 슬롯당 시간이 **1.5초 미만**이면 경고(시간 칸 범위 ÷ 슬롯 수).
  슬롯을 줄이거나 비트 시간을 늘리도록.
- **평탄함**: 콘티 전체에 `⏸` 정적이 **0개**이고 `↺` 콜백도 **0개**면 경고 — 호흡과
  수미상관 없이 나레이션만 나열된 콘티다.

### 2. 이미지 프롬프트 lint (shot 표, source_kind=image)

- **글자를 부르는 프롬프트**: text / letters / numbers / label(ed) / word / sentence /
  title / "diagram with annotations" / 연도·퍼센트 표기 류가 프롬프트에 있는데 끝에
  "absolutely no text, letters, numbers or labels" 류 봉인이 **없으면** 검출. 봉인을 더하거나
  글자 없이 푸는 구도를 제안하고, 글자가 꼭 필요하면 `❝` 분리를 제안.
- **재등장 피사체 어휘 불일치**: 같은 피사체가 여러 `▶` shot에 나오는데 명사구가 shot마다
  다르면(예: "graphics cards" ↔ "GPU board") 검출 — 한 명사구로 통일해 복사. "the same X
  schematic"처럼 **다른 shot을 참조하는 표현도 검출**(이미지 모델은 다른 shot을 못 본다 —
  명사구 원문을 그대로 다시 쓴다).
- **스타일 바이블 충돌**: photorealistic / photo of / cinematic still 등 바이블과 다른 렌더
  지시, 바이블이 "no human figures"인데 인물을 부르는 표현 등.

### 3. 텍스트카드(❝) 오남용

렌더러는 **중앙 정렬 타이포 한 덩어리**만 그린다. `❝`의 문구/의도에 순위표·막대그래프·
표·대비(흐림 vs 또렷)·다단 레이아웃이 있으면 **critical** — 그 장면은 `image`(정적 도식)
또는 `remotion`/`hyperframe`(움직이는 도식)으로 옮기도록 제안. `❝`는 인용·정의·한 줄
선언처럼 "글자 자체가 화면"인 곳만 통과.

### 4. 사실/추측 구분

- 사고실험·상상·시뮬레이션 구간이 **사실처럼 단정**되는지("~다"로 끝나는 가정 결과 등) 본다.
- 가정으로 진입하는 비트에 **진입 표시**(❝ 카드 "만약 ~라면 —" 또는 자막 표기)가 있는지 확인.
  없으면 표시 추가를 제안 — 시청자가 시뮬레이션을 관측 사실로 오해하지 않게.
- 숫자·연도·고유명사 중 의심스러운 것은 **고치지 말고** "사용자 확인 필요" 목록에만 올린다.
  대본의 주장 자체를 수정하는 것은 이 스킬의 권한 밖이다.

### 5. 자막·TTS

- **자막 길이**: 자막 한 칸이 **약 40자 초과**면 경고 — 두 비트로 나누거나 압축 제안.
- **TTS 위험 표기**: `나레이션 (TTS)` 칸의 영문 약어(top-5, RNN, BLEU…), 단위 붙은
  숫자(3.5일, 10%p…), 특수기호(괄호 병기, →, ·)는 TTS가 틀리기 쉽다 — **한글 발음 표기**를
  제안(예: top-5 → "톱 파이브", BLEU → "블루"). 자막 칸은 원표기를 유지해도 된다.
  script-to-conti가 변환 시점에 1차 적용하므로(§나레이션(TTS) 표기) 여기서는 **누락분을
  잡는 2차 안전망**이다 — 발견되면 변환 단계가 놓친 것이니 함께 보고.

### 6. CARDS.json

`❝` shot이 하나라도 있는데 CARDS.json(CONTI.md 옆 또는 generated_project/)이 없거나 해당
shot id 키가 없으면 **critical** — 화면에 shot id 문자열이 그대로 렌더된다. 누락된 키와
콘티의 `프롬프트 / 문구` 칸을 근거로 한 최종 문구 초안을 함께 제시. 새 문구는 CONTI.md 옆
CARDS.json에 쓴다(재실행에도 유지).

## 출력 형식

심각도별 표 하나로 보고한다. **critical** = 그대로 빌드하면 화면이 깨짐(§3, §6) ·
**major** = 품질이 눈에 띄게 떨어짐(글자 유발 프롬프트, 홀드 정체, TTS 오발음) ·
**minor** = 개선 여지(자막 길이, 평탄한 리듬).

```markdown
| 심각도 | 위치 | 문제 | 수정안 |
|--------|------|------|--------|
| critical | s06_errorbar | ❝에 막대그래프 레이아웃 — 렌더 불가 | source_kind=image로 변경, 프롬프트: … |
| major | b044 | 나레이션 "top-5" TTS 오발음 위험 | "톱 파이브"로 표기 (자막은 유지) |
| minor | b002 | ▦ 4슬롯 / 4초 = 1.0초/슬롯 | 슬롯 3개로 축소 또는 비트 연장 |
```

표 다음에 **"사용자 확인 필요"** 목록을 분리해 적는다 — §4의 사실 의심 항목, 의도였을 수도
있는 연출(예: 일부러 긴 홀드) 등 판단을 사용자에게 넘길 것들.

## 수정 절차

1. 보고만 먼저. **콘티를 직접 고치는 것은 사용자 승인 후**다.
2. 승인된 항목만 CONTI.md / CARDS.json에 반영한다. 나레이션 원문 변경은 §5 발음 표기처럼
   사용자가 명시 승인한 경우만.
3. 고친 뒤 반드시 재검증한다:
   ```bash
   cd <project> && weft conti   # 위반 0 유지 확인
   ```
   수정이 새 위반을 만들면(cover 범위·몽타주 슬롯 등) 그 자리에서 되돌리거나 고친다.
